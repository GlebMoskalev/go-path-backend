# Каналы

Канал — механизм для безопасной передачи данных между горутинами. Девиз Go: «не общайся через разделяемую память, разделяй память через общение».

## Создание каналов

```go
ch := make(chan int)           // небуферизованный
ch := make(chan int, 10)       // буферизованный, ёмкость 10
ch := make(chan string)        // канал строк
ch := make(chan []byte, 5)     // канал слайсов байт
```

## Отправка и получение

```go
ch <- value   // отправить value в канал
value := <-ch // получить из канала
<-ch          // получить и отбросить
```

---

## Небуферизованный канал

Отправитель **блокируется** пока получатель не заберёт значение. Это встроенная синхронизация:

```go
func main() {
    ch := make(chan int)

    go func() {
        fmt.Println("горутина: вычисляю...")
        time.Sleep(100 * time.Millisecond)
        ch <- 42  // отправляем результат
    }()

    result := <-ch  // main блокируется здесь до получения
    fmt.Println("результат:", result)
}
// горутина: вычисляю...
// результат: 42
```

Небуферизованный канал гарантирует что отправитель знает: данные получены.

## Буферизованный канал

Отправитель блокируется только когда буфер **полон**. Получатель блокируется только когда буфер **пуст**:

```go
ch := make(chan int, 3)  // буфер на 3 элемента

ch <- 1  // не блокируется
ch <- 2  // не блокируется
ch <- 3  // не блокируется
ch <- 4  // БЛОКИРУЕТСЯ — буфер полон

fmt.Println(len(ch))  // текущее количество элементов
fmt.Println(cap(ch))  // ёмкость буфера
```

Практическое применение — ограничение параллелизма (семафор):

```go
sem := make(chan struct{}, 5)  // максимум 5 одновременных операций

for _, task := range tasks {
    sem <- struct{}{}  // занять слот
    go func(t Task) {
        defer func() { <-sem }()  // освободить слот при завершении
        process(t)
    }(task)
}

// Ждать завершения всех:
for i := 0; i < cap(sem); i++ {
    sem <- struct{}{}
}
```

## Закрытие канала

`close(ch)` сигнализирует что больше значений не будет:

```go
ch := make(chan int)

go func() {
    for i := 0; i < 5; i++ {
        ch <- i
    }
    close(ch)  // сигнал окончания
}()

// Чтение до закрытия:
for v := range ch {  // автоматически завершится при close
    fmt.Println(v)
}
// 0 1 2 3 4
```

### Проверка закрытия через comma-ok

```go
v, ok := <-ch
if !ok {
    fmt.Println("канал закрыт")
}
```

### Правила закрытия

- Закрывать может только **отправитель**
- Закрытый канал можно читать — получишь оставшиеся буферизованные значения, затем нулевые значения
- Отправка в закрытый канал — паника
- Закрытие уже закрытого канала — паника

```go
ch := make(chan int, 3)
ch <- 1
ch <- 2
close(ch)

fmt.Println(<-ch)        // 1
fmt.Println(<-ch)        // 2
v, ok := <-ch
fmt.Println(v, ok)       // 0 false — канал пуст и закрыт

// ch <- 3  // panic: send on closed channel
```

## Направленные каналы

Тип канала можно ограничить: только для отправки или только для получения:

```go
var sendOnly chan<- int  // только отправка
var recvOnly <-chan int  // только получение
```

Используются в сигнатурах функций для документирования намерений и предотвращения ошибок:

```go
// producer: только отправляет
func producer(out chan<- int) {
    for i := 0; i < 5; i++ {
        out <- i
    }
    close(out)
    // <-out  // ОШИБКА компиляции: receive from send-only channel
}

// consumer: только получает
func consumer(in <-chan int) {
    for v := range in {
        fmt.Println(v)
    }
    // in <- 5  // ОШИБКА компиляции: send to receive-only channel
}

func main() {
    ch := make(chan int)
    go producer(ch)   // двунаправленный автоматически конвертируется в chan<-
    consumer(ch)      // и в <-chan
}
```

## Каналы как сигналы

Часто канал используется не для передачи данных, а как сигнал:

```go
// Сигнал завершения:
done := make(chan struct{})  // struct{} — нулевой размер

go func() {
    doWork()
    close(done)  // сигнал что работа выполнена
}()

<-done  // ждать сигнала
fmt.Println("работа завершена")
```

`chan struct{}` вместо `chan bool` — семантически точнее (нет значения, только сигнал) и не занимает лишней памяти.

---

## Практический пример: конвейер

```go
package main

import "fmt"

func generate(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums {
            out <- n
        }
        close(out)
    }()
    return out
}

func square(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        for n := range in {
            out <- n * n
        }
        close(out)
    }()
    return out
}

func main() {
    // Конвейер: генерация → возведение в квадрат
    c := generate(2, 3, 4, 5)
    out := square(c)

    for v := range out {
        fmt.Println(v)
    }
    // 4 9 16 25
}
```

## Итог

- Небуферизованный канал: синхронный, отправитель ждёт получателя
- Буферизованный канал: асинхронный до заполнения буфера
- `close(ch)` — только отправитель; сигнализирует конец данных
- `for v := range ch` — читать до закрытия
- `v, ok := <-ch` — comma-ok для проверки закрытия
- `chan<- T` (только отправка), `<-chan T` (только получение) — в сигнатурах функций
- `chan struct{}` — сигнальный канал без данных
