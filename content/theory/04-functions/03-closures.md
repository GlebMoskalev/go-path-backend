# Замыкания

Замыкание (closure) — функция, которая «захватывает» переменные из окружающего scope. Это мощный инструмент, но с неочевидным поведением, которое порождает классическую ловушку в циклах.

## Что такое замыкание

```go
func makeCounter() func() int {
    count := 0  // переменная во внешней функции
    return func() int {
        count++  // замыкание захватывает count
        return count
    }
}

func main() {
    counter := makeCounter()
    fmt.Println(counter())  // 1
    fmt.Println(counter())  // 2
    fmt.Println(counter())  // 3

    counter2 := makeCounter()
    fmt.Println(counter2())  // 1 — независимый счётчик
    fmt.Println(counter())   // 4 — первый продолжает
}
```

Каждый вызов `makeCounter()` создаёт **новое замыкание** со своей отдельной переменной `count`. Переменная живёт, пока жива функция-замыкание.

## Захват переменных по ссылке

Замыкание захватывает **переменную**, а не её значение. Это означает, что замыкание видит текущее значение переменной, а не то, которое было при создании замыкания:

```go
func main() {
    x := 10
    f := func() int {
        return x  // захвачена переменная x, не значение 10
    }

    fmt.Println(f())  // 10

    x = 20  // изменяем x после создания замыкания
    fmt.Println(f())  // 20 — f видит новое значение!
}
```

## Ловушка захвата переменной цикла

Это самая частая ошибка с замыканиями в Go (и других языках с замыканиями).

### Проблема

```go
package main

import "fmt"

func main() {
    funcs := make([]func(), 5)

    for i := 0; i < 5; i++ {
        funcs[i] = func() {
            fmt.Println(i)  // захвачена переменная i
        }
    }

    // Выполняем все функции:
    for _, f := range funcs {
        f()
    }
}
```

Что выведет этот код? Не 0, 1, 2, 3, 4 — выведет:

```
5
5
5
5
5
```

**Почему?** Все замыкания захватили одну и ту же переменную `i`. К моменту вызова функций цикл завершился, `i == 5`. Все функции читают это актуальное значение.

### Решение 1: локальная копия переменной

```go
for i := 0; i < 5; i++ {
    i := i  // создаём новую переменную i в этой итерации!
    funcs[i] = func() {
        fmt.Println(i)  // захватывает локальную копию
    }
}
// 0 1 2 3 4
```

`i := i` — не опечатка. Создаётся новая переменная с тем же именем, которая принимает текущее значение внешнего `i`. Замыкание захватывает эту новую переменную.

### Решение 2: передача через параметр

```go
for i := 0; i < 5; i++ {
    func(i int) {  // i передаётся по значению
        funcs[i] = func() {
            fmt.Println(i)
        }
    }(i)
}
// 0 1 2 3 4
```

### Решение 3 (начиная с Go 1.22): переменная цикла по умолчанию уникальна

Начиная с Go 1.22, поведение изменилось: для `for` с `:=` каждая итерация получает **отдельную копию переменной**. В старом коде (`for i := 0; ...`) это работает автоматически:

```go
// Go 1.22+: для range-цикла каждая итерация имеет свою переменную
for i := range 5 {
    funcs[i] = func() {
        fmt.Println(i)  // захватывает отдельную i
    }
}
// 0 1 2 3 4 (в Go 1.22+)
```

> Если ты используешь Go ниже 1.22, всегда применяй явное создание копии.

## Практические применения замыканий

### Мидлвар / декоратор функции

```go
func withLogging(fn func(int) int) func(int) int {
    return func(x int) int {
        fmt.Printf("вызов с %d... ", x)
        result := fn(x)
        fmt.Printf("результат: %d\n", result)
        return result
    }
}

double := func(x int) int { return x * 2 }
loggedDouble := withLogging(double)

loggedDouble(5)
// вызов с 5... результат: 10
```

### Функция с состоянием

```go
func makeAdder(base int) func(int) int {
    return func(x int) int {
        return base + x
    }
}

add5 := makeAdder(5)
add10 := makeAdder(10)

fmt.Println(add5(3))   // 8
fmt.Println(add10(3))  // 13
```

### Ленивая инициализация (memoization)

```go
func memoize(f func(int) int) func(int) int {
    cache := make(map[int]int)
    return func(n int) int {
        if v, ok := cache[n]; ok {
            return v
        }
        result := f(n)
        cache[n] = result
        return result
    }
}

var fib func(int) int
fib = memoize(func(n int) int {
    if n <= 1 {
        return n
    }
    return fib(n-1) + fib(n-2)
})

fmt.Println(fib(40))  // быстро, потому что значения кэшируются
```

### Отложенная очистка ресурсов

```go
func withTimeout(timeout time.Duration, f func()) {
    done := make(chan struct{})
    go func() {
        f()
        close(done)
    }()

    select {
    case <-done:
        fmt.Println("завершено вовремя")
    case <-time.After(timeout):
        fmt.Println("таймаут!")
    }
}
```

## Типичные ошибки

**Ошибка 1**: Захват переменной цикла (рассмотрели выше).

**Ошибка 2**: Неожиданный шаринг состояния между замыканиями.

```go
// Все горутины меняют одну и ту же переменную — race condition!
total := 0
for i := 0; i < 10; i++ {
    go func() {
        total++  // небезопасно без синхронизации!
    }()
}
```

> 💡 Race conditions и синхронизацию разберём в главе 10 «Конкурентность».

**Ошибка 3**: Утечка памяти через долгоживущие замыкания.

```go
func leakyHandler() func() {
    hugeData := make([]byte, 100*1024*1024)  // 100 MB
    return func() {
        // hugeData захвачено и будет жить пока жива эта функция
        fmt.Println(len(hugeData))
    }
}
```

Замыкание удерживает всё захваченное в памяти. Если замыкание живёт долго (например, как обработчик событий), убедись, что оно не держит ненужные большие объекты.

## Итог

- Замыкание — функция, захватывающая переменные из внешнего scope
- Захватывается **переменная**, а не её значение на момент создания
- Классическая ловушка: замыкание в цикле захватывает одну переменную цикла
- Решение: `i := i` создаёт локальную копию, или используй параметр замыкания
- Go 1.22+ исправил поведение для range-циклов
- Замыкания — основа для паттернов: декораторы, кэширование, функции с состоянием
