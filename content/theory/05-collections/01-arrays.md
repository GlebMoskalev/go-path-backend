# Массивы

Массивы в Go используются редко — в большинстве случаев используют слайсы. Но понимание массивов необходимо, потому что слайсы построены поверх них.

## Объявление массивов

```go
var a [5]int             // массив из 5 int, всё заполнено нулями
b := [3]string{"a", "b", "c"}  // массив из 3 строк с начальными значениями
c := [...]int{1, 2, 3, 4}      // компилятор считает количество: [4]int
```

`[5]int` и `[3]int` — **разные типы**. Это принципиально: размер — часть типа массива.

```go
var x [5]int
var y [3]int
x = y  // ОШИБКА: cannot use y (type [3]int) as type [5]int
```

### Доступ к элементам

```go
a := [5]int{10, 20, 30, 40, 50}

fmt.Println(a[0])  // 10
fmt.Println(a[4])  // 50
a[2] = 99
fmt.Println(a)     // [10 20 99 40 50]
fmt.Println(len(a))  // 5
```

Выход за границы — паника в runtime:

```go
fmt.Println(a[10])  // panic: index out of range [10] with length 5
```

## Массив — значение, не ссылка

Это ключевое отличие от большинства языков. В Go массив — **значение**. Присваивание и передача в функцию **копирует весь массив**:

```go
a := [3]int{1, 2, 3}
b := a  // копия!

b[0] = 99
fmt.Println(a)  // [1 2 3] — a не изменился
fmt.Println(b)  // [99 2 3]
```

При передаче в функцию тоже копируется:

```go
func modify(arr [3]int) {
    arr[0] = 99  // изменяем копию
}

a := [3]int{1, 2, 3}
modify(a)
fmt.Println(a)  // [1 2 3] — не изменился!
```

Для изменения оригинала нужен **указатель**:

```go
func modifyPtr(arr *[3]int) {
    arr[0] = 99  // изменяем оригинал через указатель
}

a := [3]int{1, 2, 3}
modifyPtr(&a)
fmt.Println(a)  // [99 2 3]
```

> 💡 Указатели подробно разберём в главе 6 «Указатели и структуры».

## Итерация по массиву

```go
nums := [5]int{10, 20, 30, 40, 50}

// С индексом и значением:
for i, v := range nums {
    fmt.Printf("nums[%d] = %d\n", i, v)
}

// Только по индексу:
for i := range nums {
    nums[i] *= 2
}
```

## Многомерные массивы

```go
// Матрица 3x3:
var matrix [3][3]int

// Инициализация:
m := [3][3]int{
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9},
}

// Доступ:
fmt.Println(m[1][2])  // 6

// Итерация:
for i, row := range m {
    for j, val := range row {
        fmt.Printf("m[%d][%d]=%d ", i, j, val)
    }
    fmt.Println()
}
```

### Практический пример: умножение матриц

```go
func multiply(a, b [3][3]int) [3][3]int {
    var result [3][3]int
    for i := range a {
        for j := range b[0] {
            for k := range b {
                result[i][j] += a[i][k] * b[k][j]
            }
        }
    }
    return result
}
```

## Сравнение массивов

Массивы сравниваемы через `==`, если элементы сравниваемые:

```go
a := [3]int{1, 2, 3}
b := [3]int{1, 2, 3}
c := [3]int{1, 2, 4}

fmt.Println(a == b)  // true
fmt.Println(a == c)  // false
```

Это делает массивы полезными как ключи в map (в отличие от слайсов).

## Когда использовать массивы

Массивы в Go используются редко. Вот реальные случаи:

**1. Известный фиксированный размер с семантической важностью:**
```go
type IPv4 [4]byte
type Color [3]uint8  // RGB

var red Color = [3]uint8{255, 0, 0}
```

**2. Ключи в map (слайсы не могут быть ключами):**
```go
type Point [2]int
visited := map[Point]bool{}
visited[Point{1, 2}] = true
```

**3. Небольшие буферы фиксированного размера:**
```go
var buf [512]byte  // стековый буфер для небольших данных
n, _ := reader.Read(buf[:])
```

**4. Криптографические хэши:**
```go
import "crypto/sha256"
hash := sha256.Sum256([]byte("hello"))  // возвращает [32]byte
```

Для всего остального — используй слайсы.

---

## Итог

- `[n]T` — массив из n элементов типа T; размер — часть типа
- Массив — **значение**: присваивание и передача в функцию копирует весь массив
- `[...]T{...}` — компилятор считает размер автоматически
- Для изменения через функцию — передавай указатель `*[n]T`
- Массивы сравниваемы через `==`
- На практике почти всегда используй слайсы — массивы нужны для специальных случаев
