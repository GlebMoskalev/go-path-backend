# Hello, World! Структура программы

Каждая история с Go начинается с одного файла. Разберём его построчно — за каждой строкой скрываются важные правила языка.

## Минимальная программа

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```

Запусти:

```bash
go run main.go
# Hello, World!
```

## Анатомия программы

### `package main`

Каждый файл Go начинается с объявления пакета. Пакет — это единица организации кода, группа связанных файлов в одной директории.

Пакет `main` особенный: он сообщает компилятору, что это исполняемая программа (а не библиотека). Без `package main` у программы нет точки входа.

Правило: **все файлы в одной директории должны принадлежать одному пакету**.

### `import "fmt"`

Импорт подключает внешние пакеты. `fmt` (сокращение от *format*) — стандартная библиотека для форматированного ввода-вывода.

Go **требует использования всех импортированных пакетов**. Если импортировал, но не использовал — ошибка компиляции. Это не придирки — это защита от мусора в коде.

```go
import "fmt"
import "os"  // если не используется os — ошибка компиляции
```

Для нескольких импортов — сгруппированный синтаксис:

```go
import (
    "fmt"
    "os"
    "strings"
)
```

### `func main()`

Точка входа программы. Когда ты запускаешь `go run` или собранный бинарник, Go ищет функцию `main` в пакете `main` и начинает выполнение с неё.

Сигнатура `main` фиксирована: она не принимает аргументов и не возвращает значений. Аргументы командной строки читаются через `os.Args`, а не через параметры функции.

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    fmt.Println("Имя программы:", os.Args[0])
    // os.Args[0] — путь к исполняемому файлу
    // os.Args[1], os.Args[2], ... — аргументы командной строки
}
```

## go run vs go build vs go install

Три команды, три разных сценария использования.

### go run — для разработки и экспериментов

```bash
go run main.go
go run .          # запустить пакет в текущей директории
go run *.go       # запустить несколько файлов
```

Компилирует во временный файл, запускает, удаляет. Никакого артефакта не остаётся. Удобно для быстрой проверки идей.

**Ограничение**: каждый раз компилирует заново. Для больших проектов это медленно.

### go build — для создания исполняемого файла

```bash
go build main.go           # создаёт ./main (или main.exe)
go build -o server main.go # задаёт имя выходного файла
go build .                 # собирает пакет в текущей директории
```

Создаёт постоянный исполняемый файл. Используй, когда нужно распространить программу, задеплоить на сервер или передать коллеге.

```bash
go build -o myapp .
./myapp
# Hello, World!
```

### go install — для установки утилит

```bash
go install .
go install github.com/some/tool@latest
```

Компилирует и кладёт бинарник в `$GOPATH/bin`. Это стандартный способ установки Go-утилит. После установки программа доступна из любой директории (если `$GOPATH/bin` в PATH).

## Практический пример: программа с несколькими файлами

В реальных проектах код разбивается на несколько файлов. Все файлы в директории принадлежат одному пакету:

```
myapp/
├── go.mod
├── main.go
└── greeting.go
```

`greeting.go`:
```go
package main

func greet(name string) string {
    return "Привет, " + name + "!"
}
```

`main.go`:
```go
package main

import "fmt"

func main() {
    message := greet("Алиса")
    fmt.Println(message)
    // Привет, Алиса!
}
```

```bash
go run .
# Привет, Алиса!
```

> 💡 Функции, типы и переменные из других файлов **того же пакета** доступны напрямую — без импорта. Подробно про пакеты и экспорт — в главе 9 «Пакеты и модули».

## Типичные ошибки новичков

**Ошибка 1**: забыть `package main` или написать другое имя пакета.

```go
// Ошибка: нет package main
import "fmt"
func main() { fmt.Println("Hi") }
// syntax error: non-declaration statement outside function body
```

**Ошибка 2**: импортировать пакет, но не использовать.

```go
package main

import (
    "fmt"
    "os"  // не используется!
)

func main() {
    fmt.Println("Hi")
}
// ./main.go:5:2: "os" imported and not used
```

**Ошибка 3**: неправильный регистр имени функции. `main` — строчными буквами, не `Main`.

## Итог

- `package main` — объявляет исполняемую программу
- `import` — подключает пакеты; неиспользуемые импорты — ошибка компиляции
- `func main()` — точка входа; фиксированная сигнатура без параметров и возвращаемых значений
- `go run` — для экспериментов, `go build` — для сборки артефакта, `go install` — для установки утилит
- Все файлы в одной директории принадлежат одному пакету
