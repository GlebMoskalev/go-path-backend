# Переменные

Go — строго типизированный язык, но при этом достаточно умный, чтобы не заставлять тебя писать тип везде. Разберём все способы объявления переменных и подводные камни каждого.

## var — явное объявление

Самая многословная, но самая явная форма:

```go
var name string
var age int
var price float64
```

Без начального значения переменная получает **нулевое значение** своего типа (подробнее ниже).

С начальным значением:

```go
var name string = "Алиса"
var age int = 30
```

Если тип можно вывести из значения — его можно опустить:

```go
var name = "Алиса"   // тип string выведен из литерала
var age = 30         // тип int
var price = 9.99     // тип float64
```

Групповое объявление — для нескольких переменных сразу:

```go
var (
    name    string = "Алиса"
    age     int    = 30
    active  bool   = true
)
```

**Когда использовать `var`:**
- На уровне пакета (вне функций) — только `var` работает
- Когда нужен конкретный тип: например `int32` вместо `int`, `float32` вместо `float64`
- Когда хочешь объявить переменную с нулевым значением

## := — короткое объявление

Самая распространённая форма внутри функций:

```go
func main() {
    name := "Алиса"
    age := 30
    price := 9.99
    active := true
}
```

Компилятор выводит тип из правой части. Это не динамическая типизация — тип фиксируется в момент объявления и не меняется.

**Ключевые ограничения `:=`:**
1. Работает только **внутри функций**
2. Слева должна быть хотя бы одна **новая** переменная

```go
// Первое объявление — обе переменные новые
data, err := readFile("a.txt")

// err уже существует, result — новая: := разрешён
result, err := saveData(data)  // ✓

// Ошибка: ни одна переменная не новая
x := 1
x := 2  // compile error: no new variables on left side of :=
```

## Нулевые значения

Это одна из лучших особенностей Go: каждый тип имеет чётко определённое нулевое значение. Переменная никогда не содержит "мусор" из памяти.

| Тип | Нулевое значение |
|-----|-----------------|
| `bool` | `false` |
| `int`, `int8`, `int16`, `int32`, `int64` | `0` |
| `uint`, `uint8`, `uint16`, `uint32`, `uint64` | `0` |
| `float32`, `float64` | `0.0` |
| `complex64`, `complex128` | `0+0i` |
| `string` | `""` (пустая строка) |
| `byte` (алиас uint8) | `0` |
| `rune` (алиас int32) | `0` |
| pointer, slice, map, channel, function | `nil` |
| struct | все поля — нулевые значения |
| array | все элементы — нулевые значения |

Практический пример:

```go
package main

import "fmt"

func main() {
    var i int
    var f float64
    var s string
    var b bool

    fmt.Printf("int: %d\n", i)       // int: 0
    fmt.Printf("float64: %f\n", f)   // float64: 0.000000
    fmt.Printf("string: %q\n", s)    // string: ""
    fmt.Printf("bool: %t\n", b)      // bool: false
}
```

Нулевые значения позволяют писать такой код безопасно:

```go
var count int
count++  // не паника, count == 0 гарантировано
fmt.Println(count)  // 1

var sb strings.Builder
sb.WriteString("hello")  // безопасно, нулевое значение Builder готово к работе
```

## Множественное присваивание

Go поддерживает присваивание нескольких переменных одновременно:

```go
a, b := 1, 2
fmt.Println(a, b)  // 1 2

// Классический swap без временной переменной:
// правая часть вычисляется полностью до присваивания
a, b = b, a
fmt.Println(a, b)  // 2 1
```

Blank identifier `_` позволяет явно проигнорировать одно из значений:

```go
value, _ := strconv.Atoi("42")  // вторым значением была бы ошибка
fmt.Println(value)              // 42
```

> 💡 Функции с несколькими возвращаемыми значениями подробно разберём в главе 4 «Функции».

## Shadowing — тень переменной

Одна из самых коварных ловушек Go, особенно в связке с `:=`.

Shadowing происходит, когда переменная во внутреннем блоке «перекрывает» переменную с тем же именем из внешнего блока:

```go
package main

import "fmt"

func main() {
    x := 10
    fmt.Println(x)  // 10

    {
        x := 20  // новая переменная x, перекрывает внешнюю
        fmt.Println(x)  // 20
    }

    fmt.Println(x)  // 10 — внешняя x не изменилась!
}
```

Это особенно опасно в конструкции `if`:

```go
func getUser(id int) (*User, error) {
    // ОШИБКА: err в блоке if — новая переменная!
    if user, err := db.Find(id); err == nil {
        return user, nil
    }
    // здесь err из блока if недоступна
    return nil, err  // compile error: undefined: err
}
```

Правильно:

```go
func getUser(id int) (*User, error) {
    user, err := db.Find(id)
    if err != nil {
        return nil, err
    }
    return user, nil
}
```

**Частый реальный баг с shadowing и `:=`:**

```go
package main

import (
    "errors"
    "fmt"
)

var ErrNotFound = errors.New("not found")

func findItem(id int) (string, error) {
    var result string
    var err error

    if id > 0 {
        result, err := lookupDB(id)  // ОШИБКА! := создаёт новые result и err
        _ = result
        _ = err
    }

    return result, err  // result и err — нулевые значения из внешнего блока!
}
```

Правильно — использовать `=` вместо `:=`:

```go
func findItem(id int) (string, error) {
    var result string
    var err error

    if id > 0 {
        result, err = lookupDB(id)  // присваиваем внешним переменным
    }

    return result, err
}
```

## Переменные на уровне пакета

Переменные можно объявлять вне функций — они принадлежат пакету:

```go
package main

import "fmt"

var (
    version = "1.0.0"
    debug   = false
)

func main() {
    fmt.Printf("Версия: %s, Отладка: %t\n", version, debug)
    // Версия: 1.0.0, Отладка: false
}
```

**Правила пакетных переменных:**
- Только `var`, не `:=`
- Инициализируются до выполнения `main()`
- Доступны из любого файла в том же пакете
- Экспортированные (с большой буквы) — из других пакетов

## Итог

| Синтаксис | Где работает | Когда использовать |
|-----------|-------------|-------------------|
| `var x T` | везде | нулевое значение, явный тип |
| `var x = v` | везде | тип выводится из значения |
| `x := v` | только в функциях | большинство случаев |
| `var (...)` | везде | группа переменных |

**Запомни:**
- Нулевые значения гарантированы — никакого неинициализированного мусора
- `:=` только внутри функций
- Shadowing: `:=` во внутреннем блоке создаёт новую переменную — внешняя остаётся без изменений
- Blank identifier `_` — для явного игнорирования значений
