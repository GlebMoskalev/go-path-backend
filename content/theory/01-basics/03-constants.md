# Константы и iota

Константы в Go — это не просто «переменные, которые нельзя менять». Система констант Go устроена интереснее, чем в большинстве языков, и понимание разницы между типизированными и нетипизированными константами избавит от неожиданных ошибок.

## Объявление констант

Базовый синтаксис:

```go
const Pi = 3.14159265358979323846
const Greeting = "Hello"
const MaxRetries = 3
const Debug = false
```

Групповое объявление:

```go
const (
    StatusOK       = 200
    StatusNotFound = 404
    StatusError    = 500
)
```

**Ключевые отличия от переменных:**
- Значение вычисляется **во время компиляции** — не в runtime
- Нельзя использовать результат вызова функции (кроме некоторых встроенных)
- Нельзя использовать `:=` для констант
- Только `bool`, числа, строки и `rune`

```go
// Нельзя:
const Now = time.Now()      // ошибка: вызов функции
const Slice = []int{1,2,3}  // ошибка: slice не может быть константой

// Можно:
const Len = len("hello")    // len — встроенная, вычисляется компилятором
const Two = 1 + 1           // арифметика констант
```

## Типизированные vs нетипизированные константы

Это самое важное и нестандартное в системе констант Go.

### Нетипизированные константы (untyped constants)

```go
const x = 42        // нетипизированная целочисленная константа
const y = 3.14      // нетипизированная константа с плавающей точкой
const s = "hello"   // нетипизированная строковая константа
```

Нетипизированная константа не привязана к конкретному типу, но при использовании принимает подходящий. Это даёт гибкость при присваивании:

```go
const x = 42

var i int = x       // x ведёт себя как int
var f float64 = x   // x ведёт себя как float64
var b byte = x      // x ведёт себя как byte

// С обычной типизированной переменной такого не получится:
var n int = 42
var f2 float64 = n  // ОШИБКА: cannot use n (type int) as type float64
```

Числовые нетипизированные константы хранятся компилятором с высокой точностью — ограничения типов применяются только в момент использования:

```go
const HugePrime = 9999999999999999999999999999999999999  // не помещается ни в int64, ни в uint64
var x float64 = HugePrime  // OK: компилятор приведёт к float64 здесь
```

### Типизированные константы

```go
const Pi float64 = 3.14159265358979323846
const MaxInt8 int8 = 127
```

Привязаны к конкретному типу и требуют явного преобразования при смешивании с другими типами:

```go
const Pi float64 = 3.14159

var x float32 = Pi  // ОШИБКА: cannot use Pi (type float64) as type float32
var y float32 = float32(Pi)  // OK
```

### Практическое правило

**Используй нетипизированные константы** для универсальных значений — математических констант, лимитов — когда они должны работать с разными типами.

**Используй типизированные константы** когда тип несёт смысл и ты хочешь защититься от случайного смешивания типов.

## iota — автоинкрементные константы

`iota` — специальный идентификатор, доступный только внутри блоков `const`. Его значение равно **порядковому номеру** объявления константы в блоке, начиная с 0.

### Простой перечень

```go
const (
    Sunday = iota  // 0
    Monday         // 1
    Tuesday        // 2
    Wednesday      // 3
    Thursday       // 4
    Friday         // 5
    Saturday       // 6
)

fmt.Println(Monday)    // 1
fmt.Println(Saturday)  // 6
```

Каждый новый блок `const` сбрасывает `iota` в 0:

```go
const (
    A = iota  // 0
    B         // 1
)

const (
    C = iota  // 0 — новый блок, iota начинается заново
    D         // 1
)
```

### Арифметика с iota

Можно использовать любое выражение, включая арифметику и битовые операции:

```go
// Степени двойки — битовые флаги
const (
    Read    = 1 << iota  // 1 << 0 = 1   (0b001)
    Write                // 1 << 1 = 2   (0b010)
    Execute              // 1 << 2 = 4   (0b100)
)

// Проверка флагов:
permissions := Read | Write  // 3 = 0b011
fmt.Println(permissions & Read != 0)    // true — есть право на чтение
fmt.Println(permissions & Execute != 0) // false — нет права на выполнение
```

```go
// Размеры файлов
const (
    _           = iota  // пропустить 0 через blank identifier
    KB float64 = 1 << (10 * iota)  // 1 << 10 = 1024
    MB                              // 1 << 20 = 1048576
    GB                              // 1 << 30
    TB                              // 1 << 40
)

fmt.Printf("1 KB = %.0f bytes\n", KB)  // 1 KB = 1024 bytes
fmt.Printf("1 MB = %.0f bytes\n", MB)  // 1 MB = 1048576 bytes
```

### Несколько значений в одной строке

```go
const (
    bit0, mask0 = 1 << iota, 1<<iota - 1  // iota=0: bit0=1, mask0=0
    bit1, mask1                            // iota=1: bit1=2, mask1=1
    bit2, mask2                            // iota=2: bit2=4, mask2=3
)
```

### Пропуск значений

```go
const (
    _  = iota  // 0 — намеренно пропущен (часто для "неверного" нулевого значения)
    One        // 1
    Two        // 2
    Three      // 3
)
```

Пропуск нуля полезен, когда нулевое значение должно означать "не задано":

```go
type Direction int

const (
    _     Direction = iota  // 0 = неопределено
    North                   // 1
    South                   // 2
    East                    // 3
    West                    // 4
)

var d Direction
fmt.Println(d == 0)  // true — переменная не инициализирована
```

## Практический пример: HTTP-статусы

```go
package main

import "fmt"

type HTTPStatus int

const (
    StatusOK                  HTTPStatus = 200
    StatusMovedPermanently    HTTPStatus = 301
    StatusBadRequest          HTTPStatus = 400
    StatusUnauthorized        HTTPStatus = 401
    StatusForbidden           HTTPStatus = 403
    StatusNotFound            HTTPStatus = 404
    StatusInternalServerError HTTPStatus = 500
)

func (s HTTPStatus) String() string {
    switch s {
    case StatusOK:
        return "200 OK"
    case StatusNotFound:
        return "404 Not Found"
    case StatusInternalServerError:
        return "500 Internal Server Error"
    default:
        return fmt.Sprintf("%d Unknown", int(s))
    }
}

func main() {
    status := StatusNotFound
    fmt.Println(status)  // 404 Not Found
}
```

> 💡 Метод `String()` делает тип совместимым с интерфейсом `fmt.Stringer` — подробнее в главе 7 «Интерфейсы».

## Константы vs переменные: когда что

**Используй константу:**
- Значение не изменится никогда (математические константы, коды статусов, конфигурационные лимиты)
- Хочешь заменить магическое число или строку понятным именем
- Нужен перечень связанных значений — аналог enum через iota

**Используй переменную:**
- Значение вычисляется в runtime
- Значение должно меняться в процессе работы программы

## Типичные ошибки

**Ошибка 1**: Попытка создать константу из runtime-значения.

```go
const start = time.Now()  // ОШИБКА: не константное выражение
var start = time.Now()    // OK
```

**Ошибка 2**: Ожидание, что iota продолжается между блоками.

```go
const A = iota  // 0
const B = iota  // 0, не 1! Каждый const — новый блок с iota=0
```

Правильно использовать один блок:

```go
const (
    A = iota  // 0
    B         // 1
)
```

---

## Итог

- Константы вычисляются во время компиляции — только числа, строки, bool
- Нетипизированные константы гибче: `const x = 42` работает как `int`, `float64`, `byte` в зависимости от контекста
- Типизированные константы строже: `const Pi float64 = 3.14` — всегда `float64`
- `iota` — элегантный способ создавать нумерованные константы (enum)
- Новый блок `const` сбрасывает `iota` в 0
