# Строки

Строки в Go устроены иначе, чем в большинстве языков. Понимание внутреннего устройства избавит от классических ошибок с Unicode и производительностью.

## Что такое строка в Go

Строка в Go — это **неизменяемая последовательность байтов**. Не символов, а именно байтов. Это принципиальная разница.

```go
s := "Привет"
fmt.Println(len(s))  // 12, а не 6!
```

Почему 12? Потому что `len()` возвращает число **байтов**, а кириллические символы занимают по 2 байта в UTF-8.

## Внутреннее устройство

Строка Go — это структура из двух полей:
- указатель на массив байтов
- длина в байтах

```text
string header:
┌──────────┬──────────┐
│  *bytes  │  length  │
└──────────┴──────────┘
      │
      ▼
┌────────────────────────────────┐
│ 48 65 6c 6c 6f (Hello = 5 байт)│
└────────────────────────────────┘
```

Строки **неизменяемы** — нельзя изменить один символ:

```go
s := "Hello"
s[0] = 'h'  // ОШИБКА: cannot assign to s[0] (strings are immutable)
```

Это сделано намеренно для безопасности и эффективности: одна строка может использоваться из нескольких мест без копирования.

## UTF-8 и Go

Исходный код Go всегда сохраняется в UTF-8, и строковые литералы — тоже:

```go
s := "Hello, 世界"  // полностью корректный Go-код
```

### Байты vs Руны

- **Байт** (`byte`, псевдоним `uint8`) — единица хранения, 0-255
- **Руна** (`rune`, псевдоним `int32`) — числовой код одного символа в Unicode (например, `'П'` = 1055)

```go
s := "Привет"

// Доступ по индексу — возвращает БАЙТ, не символ
fmt.Printf("%x\n", s[0])  // d0 — первый байт буквы 'П' в UTF-8

// Правильный доступ к символам — через range:
for i, r := range s {
    fmt.Printf("позиция %d: %c (U+%04X)\n", i, r, r)
}
// позиция 0: П (U+041F)
// позиция 2: р (U+0440)
// позиция 4: и (U+0438)
// ...

// Обрати внимание: позиции 0, 2, 4 — не 0, 1, 2!
// Каждая кириллическая буква занимает 2 байта
```

### len vs RuneCountInString

```go
import "unicode/utf8"

s := "Hello, 世界"

fmt.Println(len(s))                        // 13 (байты)
fmt.Println(utf8.RuneCountInString(s))     // 9 (символы)

// Конвертация в []rune для посимвольного доступа:
runes := []rune(s)
fmt.Println(len(runes))  // 9
fmt.Println(string(runes[7]))  // 世
```

## Операции со строками

### Конкатенация

```go
s1 := "Hello"
s2 := "World"

// Простая конкатенация — создаёт новую строку:
result := s1 + ", " + s2 + "!"
fmt.Println(result)  // Hello, World!
```

**Проблема**: каждый `+` создаёт новую строку в памяти. Для 3-5 строк это нормально. Для цикла с сотнями конкатенаций — катастрофа.

```go
// МЕДЛЕННО: O(n²) аллокаций
var result string
for i := 0; i < 1000; i++ {
    result += fmt.Sprintf("item%d ", i)  // каждый раз новая строка!
}
```

### strings.Builder — эффективная конкатенация

```go
import (
    "fmt"
    "strings"
)

var sb strings.Builder

for i := 0; i < 1000; i++ {
    fmt.Fprintf(&sb, "item%d ", i)  // пишет в буфер
}

result := sb.String()  // финальная строка создаётся один раз
```

`strings.Builder` накапливает данные в буфере и создаёт строку только при вызове `.String()`. Внутри — `[]byte`, который растёт по мере необходимости.

Когда заранее известен примерный размер:

```go
var sb strings.Builder
sb.Grow(1024)  // выделить буфер заранее
```

### Сравнение строк

```go
s1 := "apple"
s2 := "banana"

fmt.Println(s1 == s2)   // false
fmt.Println(s1 < s2)    // true (лексикографически)
fmt.Println(s1 != s2)   // true

// Без учёта регистра:
import "strings"
fmt.Println(strings.EqualFold("Go", "go"))  // true
```

## Пакет strings — основные функции

```go
import "strings"

s := "Hello, World!"

// Поиск
strings.Contains(s, "World")      // true
strings.HasPrefix(s, "Hello")     // true
strings.HasSuffix(s, "!")         // true
strings.Index(s, "World")         // 7
strings.Count(s, "l")             // 3

// Преобразование
strings.ToUpper(s)                // "HELLO, WORLD!"
strings.ToLower(s)                // "hello, world!"
strings.Title(s)                  // устарело, используй golang.org/x/text

// Удаление пробелов
strings.TrimSpace("  hello  ")    // "hello"
strings.Trim("***hello***", "*")  // "hello"
strings.TrimLeft("  hi", " ")     // "hi"

// Разбивка и соединение
parts := strings.Split("a,b,c", ",")  // ["a", "b", "c"]
joined := strings.Join(parts, " - ")   // "a - b - c"

fields := strings.Fields("  foo   bar  ")  // ["foo", "bar"]

// Замена
strings.Replace(s, "World", "Go", 1)  // "Hello, Go!" (1 замена)
strings.ReplaceAll(s, "l", "L")        // "HeLLo, WorLd!"

// Повторение
strings.Repeat("ab", 3)  // "ababab"
```

## Преобразование между string, []byte и []rune

```go
s := "Привет"

// string → []byte
b := []byte(s)
fmt.Println(b)  // [208 159 209 128 208 184 208 178 208 181 209 130]

// []byte → string
s2 := string(b)
fmt.Println(s2)  // Привет

// string → []rune
r := []rune(s)
fmt.Println(len(r))  // 6 (символы)

// []rune → string
s3 := string(r)
fmt.Println(s3)  // Привет

// Изменение строки через []byte:
b2 := []byte(s)
b2[0] = 'P'  // ОСТОРОЖНО: меняет только первый байт
// Кириллица занимает 2 байта, поэтому замена одного байта ломает символ
fmt.Println(string(b2))  // вместо 'П' получится мусор
```

**Важно**: при преобразовании `string` ↔ `[]byte` или `string` ↔ `[]rune` данные **копируются**. Это защищает неизменяемость строки.

## Строковые литералы

Go поддерживает два вида строковых литералов:

### Interpreted (в двойных кавычках)
Обрабатывают escape-последовательности:

```go
s1 := "Hello\tWorld\n"  // \t = tab, \n = newline
s2 := "Кавычки: \"текст\""
s3 := "\u041F\u0440\u0438\u0432\u0435\u0442"  // "Привет" через Unicode
```

### Raw (в обратных кавычках)
Буквальный текст, без обработки escape:

```go
path := `C:\Users\Alice\Documents`  // обратный слеш не нужно экранировать
json := `{"name": "Alice", "age": 30}`
regex := `\d+\.\d+`  // регулярное выражение без двойного экранирования

sql := `
    SELECT *
    FROM users
    WHERE active = true
    ORDER BY name
`
```

Raw-строки незаменимы для JSON, SQL, путей Windows, регулярных выражений.

## Типичные ошибки

**Ошибка 1**: Посимвольный обход через индекс.

```go
s := "Привет"
for i := 0; i < len(s); i++ {
    fmt.Printf("%c", s[i])  // Ð ½Ñ Ð¸Ð²Ðµ... — мусор!
}

// Правильно — через range:
for _, r := range s {
    fmt.Printf("%c", r)  // Привет
}
```

**Ошибка 2**: Попытка изменить строку напрямую.

```go
s := "Hello"
s[0] = 'h'  // compile error: cannot assign to s[0]

// Правильно:
b := []byte(s)
b[0] = 'h'
s = string(b)
fmt.Println(s)  // hello
```

**Ошибка 3**: Использование `len()` для подсчёта символов.

```go
s := "Hello, 世界"
if len(s) > 8 {  // 13 > 8, хотя символов только 9
    // ...
}

// Правильно:
if utf8.RuneCountInString(s) > 8 {
    // ...
}
```

## Итог

- Строка — неизменяемая последовательность байтов, не символов
- `len(s)` — байты, `utf8.RuneCountInString(s)` — символы
- `for range` итерирует по рунам (символам), индексация `s[i]` — по байтам
- `string` ↔ `[]byte` ↔ `[]rune` — все конвертации копируют данные
- Для эффективной конкатенации в цикле — `strings.Builder`
- Raw-строки (обратные кавычки) — для многострочных литералов и путей
