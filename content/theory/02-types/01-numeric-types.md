# Числовые типы

Go предоставляет богатый набор числовых типов с явными размерами. В отличие от C, где `int` может быть 16, 32 или 64 бита в зависимости от платформы, Go даёт полный контроль.

## Целочисленные типы

### Знаковые (signed) — хранят отрицательные и положительные числа

| Тип | Размер | Диапазон |
|-----|--------|----------|
| `int8` | 8 бит | -128 … 127 |
| `int16` | 16 бит | -32 768 … 32 767 |
| `int32` | 32 бита | -2 147 483 648 … 2 147 483 647 |
| `int64` | 64 бита | -9 223 372 036 854 775 808 … 9 223 372 036 854 775 807 |

### Беззнаковые (unsigned) — только положительные числа

| Тип | Размер | Диапазон |
|-----|--------|----------|
| `uint8` | 8 бит | 0 … 255 |
| `uint16` | 16 бит | 0 … 65 535 |
| `uint32` | 32 бита | 0 … 4 294 967 295 |
| `uint64` | 64 бита | 0 … 18 446 744 073 709 551 615 |

### Платформозависимые типы

| Тип | Размер | Использование |
|-----|--------|---------------|
| `int` | 32 или 64 бита (зависит от платформы) | Общий целочисленный тип |
| `uint` | 32 или 64 бита | Беззнаковый общий тип |
| `uintptr` | достаточно для хранения указателя | Низкоуровневое программирование |

### Псевдонимы

```go
byte  // псевдоним для uint8 — для работы с байтами/бинарными данными
rune  // псевдоним для int32 — для Unicode-символов
```

### Какой тип использовать?

```go
// Общее правило: используй просто int, если нет причин иначе
var count int = 42
var index int = 0

// int8/int16 — когда важна экономия памяти (большие массивы)
var scores [1000000]int8

// int32/int64 — когда важен конкретный размер (сериализация, протоколы)
var timestamp int64 = time.Now().Unix()

// uint8 == byte — для работы с бинарными данными
var buf []byte

// uint32 — IPv4 адрес, цвет RGBA
var color uint32 = 0xFF0000FF
```

## Числа с плавающей точкой

```go
var f32 float32 = 3.14
var f64 float64 = 3.141592653589793

fmt.Printf("float32: %.10f\n", f32)  // float32: 3.1400001049
fmt.Printf("float64: %.15f\n", f64)  // float64: 3.141592653589793
```

- `float32` — 32 бита, ~7 значащих цифр, IEEE 754
- `float64` — 64 бита, ~15 значащих цифр, IEEE 754

**По умолчанию используй `float64`** — он точнее, и на большинстве платформ не медленнее `float32`.

### Специальные значения

```go
import "math"

posInf := math.Inf(1)   // +Infinity
negInf := math.Inf(-1)  // -Infinity
nan := math.NaN()        // Not a Number

fmt.Println(posInf)            // +Inf
fmt.Println(math.IsInf(posInf, 1))  // true
fmt.Println(math.IsNaN(nan))         // true

// ВАЖНО: NaN != NaN
fmt.Println(nan == nan)  // false — это нормальное поведение IEEE 754
```

### Точность и сравнение float

Никогда не сравнивай float на равенство через `==`:

```go
a := 0.1 + 0.2
b := 0.3

fmt.Println(a == b)       // false! (0.1+0.2 = 0.30000000000000004)
fmt.Println(a)            // 0.30000000000000004

// Правильное сравнение — через эпсилон:
const epsilon = 1e-9
fmt.Println(math.Abs(a-b) < epsilon)  // true
```

## Комплексные числа

Go — один из немногих языков с встроенной поддержкой комплексных чисел:

```go
var c1 complex64  = 1 + 2i
var c2 complex128 = 3.14 + 2.71i

c3 := complex(1.0, 2.0)  // создание через встроенную функцию
fmt.Println(real(c3))    // 1
fmt.Println(imag(c3))    // 2

// Арифметика:
sum := c1 + complex64(3+4i)
fmt.Println(sum)  // (4+6i)
```

На практике комплексные числа используются в научных вычислениях — в большинстве бизнес-приложений ты их не встретишь.

## Переполнение (overflow)

**Критически важно:** Go **не паникует** при переполнении целых чисел. Значение просто "оборачивается":

```go
package main

import "fmt"

func main() {
    var x int8 = 127
    x++
    fmt.Println(x)  // -128 (переполнение!)

    var u uint8 = 255
    u++
    fmt.Println(u)  // 0 (переполнение!)
}
```

Это поведение называется **wrap-around** и унаследовано от машинной арифметики. Компилятор не предупреждает об этом — ответственность на программисте.

**Как проверить переполнение:**

```go
import "math"

func safeAdd(a, b int8) (int8, error) {
    if b > 0 && a > math.MaxInt8-b {
        return 0, errors.New("overflow")
    }
    if b < 0 && a < math.MinInt8-b {
        return 0, errors.New("underflow")
    }
    return a + b, nil
}
```

## Арифметические операции

```go
a, b := 10, 3

fmt.Println(a + b)   // 13 — сложение
fmt.Println(a - b)   // 7  — вычитание
fmt.Println(a * b)   // 30 — умножение
fmt.Println(a / b)   // 3  — целочисленное деление (остаток отбрасывается!)
fmt.Println(a % b)   // 1  — остаток от деления

// Деление float:
x, y := 10.0, 3.0
fmt.Println(x / y)   // 3.3333333333333335

// Целочисленное деление — частая ловушка:
fmt.Println(5 / 2)    // 2, не 2.5!
fmt.Println(5.0 / 2)  // 2.5 — хоть один float — результат float
```

### Побитовые операции

```go
a, b := 0b1010, 0b1100  // 10 и 12 в двоичном

fmt.Printf("%04b\n", a & b)   // 1000 — AND
fmt.Printf("%04b\n", a | b)   // 1110 — OR
fmt.Printf("%04b\n", a ^ b)   // 0110 — XOR
fmt.Printf("%04b\n", a &^ b)  // 0010 — AND NOT (bit clear)
fmt.Printf("%04b\n", a << 1)  // 10100 — сдвиг влево
fmt.Printf("%04b\n", a >> 1)  // 0101  — сдвиг вправо
```

## Константы math.MaxInt и math.MinInt

Пакет `math` содержит константы для граничных значений:

```go
import "math"

fmt.Println(math.MaxInt8)   // 127
fmt.Println(math.MinInt8)   // -128
fmt.Println(math.MaxInt16)  // 32767
fmt.Println(math.MaxInt32)  // 2147483647
fmt.Println(math.MaxInt64)  // 9223372036854775807
fmt.Println(math.MaxInt)    // зависит от платформы (64-bit: 9223372036854775807)

fmt.Println(math.MaxFloat32)  // 3.4028234663852886e+38
fmt.Println(math.MaxFloat64)  // 1.7976931348623157e+308
```

## Типичные ошибки

**Ошибка 1**: Смешивание типов без явного преобразования.

```go
var a int32 = 10
var b int64 = 20
fmt.Println(a + b)  // ОШИБКА: mismatched types int32 and int64

fmt.Println(int64(a) + b)  // OK: 30
```

**Ошибка 2**: Потеря точности при преобразовании float в int.

```go
f := 3.99
i := int(f)  // 3 — дробная часть просто отрезается, не округляется!
fmt.Println(i)  // 3

// Если нужно округление:
import "math"
rounded := int(math.Round(f))  // 4
```

**Ошибка 3**: Целочисленное деление там, где ожидалось дробное.

```go
func average(total, count int) float64 {
    return float64(total) / float64(count)  // правильно
    // return total / count  // неправильно — потеря дробной части
}
```

---

## Итог

- Используй `int` для обычных целых чисел, `float64` для дробных
- Типы с явными размерами (`int32`, `uint8`) — для сериализации, протоколов, экономии памяти
- Переполнение целых чисел не вызывает панику — "оборачивается" молча
- Никогда не сравнивай float через `==` — используй epsilon-сравнение
- При смешивании типов — явное преобразование обязательно
