# Модели данных

В архитектуре Go-приложений **модели** — это структуры данных, которые описывают сущности вашего приложения. Они используются всеми слоями: хранилищем, сервисом и обработчиками HTTP.

## Что нужно реализовать

Создайте три структуры в пакете `model`:

### 1. `Todo` — основная сущность задачи

| Поле        | Тип         | JSON-тег       | Описание              |
|-------------|-------------|----------------|-----------------------|
| ID          | `int`       | `"id"`         | Уникальный идентификатор |
| Title       | `string`    | `"title"`      | Название задачи       |
| Description | `string`    | `"description"`| Описание              |
| Done        | `bool`      | `"done"`       | Статус выполнения     |
| CreatedAt   | `time.Time` | `"created_at"` | Дата создания         |

### 2. `CreateTodoRequest` — запрос на создание

| Поле        | Тип      | JSON-тег       |
|-------------|----------|----------------|
| Title       | `string` | `"title"`      |
| Description | `string` | `"description"`|

### 3. `UpdateTodoRequest` — запрос на обновление

Здесь используются **указатели**, чтобы отличать «поле не передано» от «установлено в пустое значение»:

| Поле        | Тип       | JSON-тег       |
|-------------|-----------|----------------|
| Title       | `*string` | `"title"`      |
| Description | `*string` | `"description"`|
| Done        | `*bool`   | `"done"`       |

## Пример использования

```go
todo := model.Todo{
    ID:    1,
    Title: "Купить молоко",
    Done:  false,
}

// Частичное обновление — меняем только статус
done := true
update := model.UpdateTodoRequest{Done: &done}
```
